# v9 报告

日期：2026-03-14

## 本轮自问自答

问题：

- 现在日报信息已经很多，为什么还要再做一层推送摘要？

回答：

- 因为完整日报适合存档和深看，不适合直接发消息。推送应该是“先看结论”，把高可信词、新增词、待复核词压缩成一屏内可读的摘要。

## 本轮目标

- 推送内容和完整日报解耦
- `WeCom` 推送改成摘要式 Markdown
- `Email` 预览增加 HTML 版本
- 保持现有 CLI 和配置方式不变

## 交付内容

- 推送摘要渲染器：`src/pushers/render-digest.ts`
- `WeCom` dry-run 现在输出摘要 Markdown JSON
- `Email` dry-run 现在输出包含 `text + html` 的预览
- 推送链路直接读取数据库中的 `summary_json`
