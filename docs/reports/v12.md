# v12 报告

日期：2026-03-14

## 本轮自问自答

问题：

- 既然已经有 `status` 命令，为什么还要继续改？

回答：

- 因为之前的 `status` 只告诉我“最新报告在哪”，不能回答需求方最关心的问题：现在有没有日报、调度装没装、推送现在能不能发。

## 本轮目标

- 把 `status` 升级成交付状态总览
- 一次性输出报告、调度、信源、推送准备度
- 保持无副作用，可在任意环境查看

## 交付内容

- 新增 `src/pipeline/status-overview.ts`
- `status` 现在会输出：
  - 最新报告摘要
  - 启用中的第一/第二信源
  - 调度产物存在情况
  - 当前运行平台下的调度状态
  - 推送通道 readiness

## 实际测试结果

已执行：

- `npm run typecheck`
- `npm test`
- 新增状态总览单元测试
- 干净目录下生成 fixture 报告和调度文件后读取状态总览

结果：

- TypeScript 类型检查通过
- `vitest` 共 35 个测试全部通过
- `status` 已能输出报告是否存在、聚合词条、高可信数量、待复核数量
- `status` 已能区分 `preview_only / ready / misconfigured`
- 干净目录验收中，状态总览已正确识别 macOS 调度产物存在、email ready、wecom preview_only
