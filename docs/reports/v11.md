# v11 报告

日期：2026-03-14

## 本轮自问自答

问题：

- 既然已经能生成调度文件，为什么还要继续做调度安装和状态查看？

回答：

- 因为从需求方角度，“给我一个 plist / ps1”不等于“我已经能每天收到推送”。必须补到安装、查看状态、移除，才算接近可交付。

## 本轮目标

- 把调度能力从“生成文件”补到“安装 / 状态 / 移除”
- 保持仓库内可测试，不直接依赖真实系统变更
- 兼顾 macOS 执行路径和 Windows 脚本路径

## 交付内容

- 新增 `schedule:install`
- 新增 `schedule:status`
- 新增 `schedule:remove`
- macOS 使用 `launchctl` 的安装、状态、移除流程
- Windows 生成安装、状态、移除 PowerShell 脚本

## 实际测试结果

已执行：

- `npm run typecheck`
- `npm test`
- 新增调度管理单元测试
- 干净目录下使用注入执行器模拟 `launchctl` 安装、状态、移除
- 验证 Windows 在非 Windows 运行时会生成脚本但不误执行

结果：

- TypeScript 类型检查通过
- `vitest` 共 33 个测试全部通过
- macOS 调度安装会写入 LaunchAgents 并调用 `launchctl`
- macOS 调度状态可返回 installed / active 结果
- macOS 调度移除会卸载并删除 plist
- Windows 在非匹配运行时会生成 install / status / remove 脚本，适合转移到 Windows 执行
