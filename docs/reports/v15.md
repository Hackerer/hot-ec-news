# v15 报告

日期：2026-03-14

## 本轮自问自答

问题：

- 既然 `status` 已经能看报告和调度状态，为什么还要继续补 pipeline run audit？

回答：

- 因为“当前配置看起来没问题”不等于“今天早上那次日跑真的成功了”。需求方真正关心的是最近一次 `run:daily` 到底导入了什么、跳过了什么、有没有告警、最终有没有出报告。

## 本轮目标

- 给 `run:daily` 增加可追溯运行记录
- 把最近一次运行结果接进 `status`
- 让日报交付从“看配置”升级到“看真实运行结果”

## 交付内容

- 新增 `pipeline_runs` 表
- `run:daily` 现在会记录：
  - `startedAt`
  - `finishedAt`
  - `warnings`
  - `importedFiles`
  - `skippedFiles`
  - `pushOutputs`
  - `reportPath`
  - `errorMessage`
- `status` 现在会输出最近一次 `run:daily` 结果

## 实际测试结果

已执行：

- `npm run typecheck`

- 新增 pipeline run history 单元测试
- 干净目录下执行一次 `run:daily` 后读取 `status`

- 干净目录验收中，`lastRun.status` 为 `success`
