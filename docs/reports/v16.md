# v16 报告

日期：2026-03-14

## 本轮自问自答

问题：

- 既然 `v15` 已经能看到最近一次运行，为什么还要继续做最近几次运行视图？

回答：

- 因为日推系统最怕“偶发成功”。需求方验收时真正想知道的是系统是不是连续稳定，而不是只看最后一次恰好成功。最近几次运行视图能直接判断是否存在连续失败、间歇失败或重复跳过。

## 本轮目标

- 把 pipeline run audit 扩成最近几次运行视图
- 提供直接可用的 `runs` CLI
- 让 `status` 不只显示最后一次，还能显示最近运行摘要

## 交付内容

- 新增 `listPipelineRuns`
- 新增 `runs --limit N` CLI 命令
- `status` 现在会输出最近几次运行摘要
- 失败运行会在 `status` 和 `runs` 中暴露 `errorMessage`

## 实际测试结果

已执行：

- `npm run typecheck`
- `npm test`
- 新增 recent runs 单元测试
- 干净目录下人工写入 success + failed 两条运行记录后执行 `runs` 与 `status`

结果：

- TypeScript 类型检查通过
- `vitest` 共 41 个测试全部通过
- `runs --limit 2` 已能按时间倒序展示最近两次运行
- `status` 已能展示最近一次失败原因和最近运行摘要
- 干净目录 CLI 验收中已真实出现 `JD unavailable` 错误展示
