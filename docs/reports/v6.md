# v6 报告

日期：2026-03-14

## 本轮自问自答

问题：

- 如果后面继续接平台、继续加推送渠道，当前代码结构会不会很快失控？

回答：

- 会。所以这轮不继续堆功能，而是把项目升级成配置驱动、注册表驱动的结构。

## 本轮目标

- 把外部项目的精髓沉淀成产品方案和技术方案
- 增加配置加载器
- 增加 collector / importer / pusher registry
- 让 `run:daily` 按配置自动执行

## 交付内容

- 产品方案文档
- 技术方案文档
- `config/app.example.json`
- `src/config/load-config.ts`
- `src/collectors/registry.ts`
- `src/importers/registry.ts`
- `src/pushers/registry.ts`

## 实际测试结果

已执行：

- `npm run typecheck`
- `npm run check`
- 干净目录下写入 `config/app.json`
- 开启 `autoPushOnDaily`
- 开启 `wecom/email` dry-run push channel
- `HOT_EC_NEWS_ROOT=/tmp/... npm run run:daily`

结果：

- TypeScript 类型检查通过
- `vitest` 共 20 个测试全部通过
- 配置驱动的 `run:daily` 成功导入第三方文件

- registry 与 config loader 已实际接入主流程
