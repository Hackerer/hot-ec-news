# v10 报告

日期：2026-03-14

## 本轮自问自答

问题：

- 现在功能已经不少了，为什么这一轮不继续加平台，而是回头修 review 里提的问题？

回答：

- 因为这几个问题会直接影响结果可信度和生产可用性。平台再多，如果重复执行会放大数据、配置写了不生效、邮件还能被脏数据打坏，那系统仍然不稳。

## 本轮目标

- 修复同日重复执行的幂等问题
- 让 `reportDir / databasePath / categories` 真正生效
- 给邮件 HTML 渲染补安全转义

## 交付内容

- live 数据按“同日主信源”覆盖写入
- third-party 数据按“同日同 provider”覆盖写入
- pipeline / CLI / push 统一走配置化路径
- `categories` 同时控制采集 seeds 和报告区块
- 邮件 HTML 对关键词和 warning 做转义

## 实际测试结果

已执行：

- `npm run typecheck`
- `npm test`
- 新增幂等测试
- 新增配置路径与类目测试
- 新增 HTML 转义测试
- 干净目录下执行两次 `run:daily`
- 干净目录下执行带自定义路径和单类目配置的 `demo:fixtures`

结果：

- TypeScript 类型检查通过
- `vitest` 共 28 个测试全部通过
- 同日连续两次执行 `run:daily` 后，当日记录数保持稳定
- 自定义 `reportDir / databasePath` 已实际生效
- 单类目配置下报告只输出目标类目区块
- 邮件 HTML 已对不可信内容做转义
