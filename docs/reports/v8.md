# v8 报告

日期：2026-03-14

## 本轮自问自答

问题：

- 已经有分数和校验状态了，为什么还要继续做可信度分层？

回答：

- 因为运营真正需要的不是“所有词”，而是“先看哪些词最可信，哪些词要人工复核”。没有这个分层，日报仍然偏技术输出，不够适合直接消费。

## 本轮目标

- 给聚合结果增加可信度分层
- 在日报里突出高可信热词
- 在日报里输出待人工复核清单
- 用明确原因解释为什么需要复核

## 交付内容

- 可信度等级：`high / medium / low`
- 复核标记：`single_source / secondary_only / new_unvalidated / low_confidence`
- 报告区块：`高可信热词`
- 报告区块：`待人工复核`
- 总览统计增加高、中、低可信和待复核数量

## 实际测试结果

已执行：

- `npm run typecheck`
- `npm test`
- 新增可信度单元测试
- 新增 Markdown 渲染单元测试
- 干净目录下执行 `run:daily` 验证报告输出

结果：

- TypeScript 类型检查通过
- `vitest` 共 23 个测试全部通过
- 高可信词会优先进入可信度清单
- 单一信源、仅第二信源、新词未校验、低可信度会进入复核队列
- 干净目录下 `run:daily` 已成功生成高可信日报
- 单一主信源场景下 `run:daily` 已成功生成非空复核队列
- 日报已能直接作为人工判断优先级的工作面板使用
